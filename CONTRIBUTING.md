# 参与 SENM

感谢你愿意继续完善 SENM。建议先阅读 [AGENTS.md](AGENTS.md)，其中记录了当前产品已经验证过的交互与 Prompt 生成原则。

## 提交问题

反馈生成质量时，请尽量附上：

- 原始剧情、画幅和时长。
- 实际 Prompt 与预期镜头关系。
- 问题属于剧情理解、人物关系、分镜节奏、模型连接、平台提醒还是界面。
- 可公开的截图或生成视频。请先移除真人隐私、API Key 和未授权素材。

## 开发流程

1. Fork 本项目并从 `main` 创建分支。
2. 使用 Node.js 20+ 执行 `npm install`。
3. 通过 `npm run dev` 启动桌面应用。
4. 保持修改聚焦，并为共享翻译逻辑补充测试。
5. 提交 Pull Request 前运行：

```bash
npm test
npm run test:credentials
npm run build
```

涉及打包、托盘或窗口行为时，请说明你实际验证过的系统与架构。不要在 Pull Request 中提交 `release/`、用户配置、密钥、签名证书或测试素材。

## Prompt 策略贡献

优秀的样例应说明“为什么这个镜头能让观众读懂关系”，而不只是罗列景别与运镜。优先贡献可以复用的戏剧规律、失败反例和小范围测试，避免把单个剧情硬编码进全局 Prompt。

## 行为准则

保持讨论针对作品与实现，不攻击创作者。涉及内容审核时，SENM 的定位是准确提醒，不替用户作价值判断，也不承诺平台一定通过审核。
